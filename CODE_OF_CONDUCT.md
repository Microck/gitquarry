# Code of Conduct

## Our Standard

Participation in this project should be straightforward, respectful, and technically serious.

Examples of expected behavior:

- discuss ideas directly without personal attacks
- assume good faith until there is evidence otherwise
- accept technical disagreement without turning it into hostility
- keep feedback specific, actionable, and relevant to the work
- respect boundaries around private credentials, personal data, and access

Examples of unacceptable behavior:

- harassment, intimidation, or threats
- slurs or demeaning language
- sexualized language or unwanted sexual attention
- doxxing or sharing private information without permission
- repeated bad-faith disruption of discussion or review

## Enforcement

Project maintainers may remove, edit, or reject comments, issues, pull requests, code, or other contributions that violate this code of conduct.

## Reporting

If you experience or observe unacceptable behavior, report it privately to `contact@micr.dev` rather than escalating in public threads.

## Scope

This code of conduct applies to:

- repository discussions
- issues and pull requests
- project documentation spaces
- other project-controlled community channels

## Consequences

Depending on severity, enforcement can include:

- a warning
- content removal
- temporary participation restrictions
- permanent removal from project spaces
